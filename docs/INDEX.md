# Documentation Index

This index explains what each documentation file is for and how to onboard safely.

## Canonical Documents (Current Source of Truth)

- `README.md`  
  Repository entry point, Version 0 scope, structure overview.
- `PROJECT_VISION.md`  
  Long-term product direction, platform targets, non-goals during Version 0.
- `ROADMAP.md`  
  Phase-based roadmap with Version 0 and Version 1 boundaries.
- `VERSION_0_AUDIT.md`  
  Audit findings, classification, and open review items.
- `DEVELOPMENT_NOTES.md`  
  Working conventions and folder intent.
- `CONTRIBUTING.md`  
  Collaboration rules and change documentation expectations.
- `docs/ARCHITECTURE_DECISIONS.md`  
  Decision log for durable technical/product direction.
- `CHANGELOG.md`  
  Chronological record of structural/documentation changes.
- `ARCHIVE_MANIFEST.md`  
  Historical mapping of moved files and preservation rationale.

## Recommended Reading Order (New Contributors)

1. `README.md`
2. `PROJECT_VISION.md`
3. `ROADMAP.md`
4. `CONTRIBUTING.md`
5. `docs/ARCHITECTURE_DECISIONS.md`
6. `VERSION_0_AUDIT.md`
7. `DEVELOPMENT_NOTES.md`
8. `ARCHIVE_MANIFEST.md` (only if you need historical movement context)

## Quick Orientation

- Active experimental implementations live in `prototypes/`.
- `src/` is reserved for future active application code.
- Do not treat archived docs as implementation instructions without reconciling with canonical docs first.
